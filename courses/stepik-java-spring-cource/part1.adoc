Инверсия контроля имеет 2 реализации:

* внреднеие зав-ти
* паттерн фабрика

Инверсия зав-ти часто реализуется через конструктор, когда зависимому объекту в конструкторе передают то, от чего он зависит и лучше как можно абстрактнее

Инверсия контроля - это вынесения процесса создания зав-ти вне объекта, которому эта зав-ть нужна.
Внедрение зав-ти - это способ как эту вынесенную зав-ть передать объекту, которому эта зав-ть нужна.
Документация Spring 4.x советует использовать внедрение через конструктор

Ошибка: Spring не знает какую реализацию для бина подтавлять когда их больше двух
Решение: @Primary аннотация позволяет указать какой класс приоритетнее использовать для внедреняя зав-ти для бина, у которого есть абстракция в зав-ти на другие классы

Spring Container - ядро спринга, которое позволяет создавать, конфигурировать и управлять объектами.
Настроенные Spring Container-ом объекты наз-ся бинами.

нужный раздел базы знаний. Поэтому исходники меняться не должны, так как это текст из источника, а данные базы знаний могут меняться, так как знания можно дополнять и редактировать.

2.10 Прокси и Spring АОП
Прокси - это шаблон проектирования, позволяющий добавить новую логику к классу, без изменения самого класса.
Spring AOP - это реализация AOP, в которой аспекты применяются к прокси-объектам. Это позволяет внедрять поведение в код без изменения его самого.
Spring AOP использует расшируенную версию Spring J.
В АОП всё основано не на объектах, а на аспектах.

* По умолчанию Spring старается использовать JDK прокси, при условии, что у обьекта есть интерфейс. В противном случае будет использован CGLIB прокси
